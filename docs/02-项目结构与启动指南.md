# 项目结构与启动指南

## 1. 项目简介

本项目是基于长安链的校园电子学生证管理系统，采用区块链技术实现身份数据的去中心化存储与可信验证。

## 2. 项目目录结构

```

│   ├── config/                   # 长安链配置

│   ├── .env.example             # 环境变量示例
```

## 3. 核心文件说明

### 3.1 后端核心文件

| 文件 | 说明 |
|------|------|
| `backend/app.js` | Express 服务入口，定义所有 API 路由 |
| `backend/chainmaker-client.js` | 长安链客户端封装，支持双模式切换 |

### 3.2 智能合约

| 文件 | 说明 |
|------|------|
| `SmartContract/did-contract.go` | DID 身份管理合约，实现身份的增删改查 |

### 3.3 前端页面

| 文件 | 说明 |
|------|------|
| `campus-system.html` | 完整的校园电子学生证管理系统 |
| `demo.html` | 简化版演示页面 |

## 4. 快速启动（5 分钟）

### 步骤 1：启动后端服务

```powershell
cd backend
npm install
node app.js
```

### 步骤 2：打开浏览器

访问：http://localhost:3000

系统将自动重定向到 `campus-system.html`

## 5. 启动验证

### 5.1 检查服务状态

访问健康检查接口：
```
http://localhost:3000/health
```

## 6. 技术栈

### 6.1 后端技术

- **框架**：Node.js + Express.js
- **区块链**：长安链 (ChainMaker)
- **存储**：本地 JSON 文件（DEMO_MODE）或链上存储（CHAIN_MODE）

### 6.2 前端技术

- **原生 HTML5 + CSS3 + JavaScript**
- **响应式布局**
- **无框架依赖**，易于理解和修改

### 6.3 智能合约

- **语言**：Go
- **运行环境**：WASM
- **SDK**：长安链合约 SDK

## 7. 运行模式

### 7.1 DEMO_MODE（默认）

- 数据存储在本地 JSON 文件
- 无需启动区块链节点
- 适合开发、测试、演示

### 7.2 CHAIN_MODE

- 数据存储在长安链上
- 需要部署链节点
- 适合生产环境

## 8. 扩展功能的集成

新增功能需要修改以下文件：

1. **后端**：`backend/chainmaker-client.js`（添加数据管理方法）
2. **API**：`backend/app.js`（添加路由）
3. **前端**：`campus-system.html`（添加页面和交互）

### 8.1 已扩展的模块

| 模块 | 说明 | 数据文件 |
|------|------|---------|
| 消费管理 | 学生消费记录 | consumption-storage.json |
| 场馆预约 | 场馆预约管理 | reservation-storage.json |
| 成就管理 | 学生成就证书 | achievement-storage.json |
