# 环境配置与部署指南

## 1. 系统要求

- **操作系统**：Windows 10/11、macOS 10.14+、Ubuntu 20.04+
- **Node.js**：16 LTS / 18 LTS / 20+
- **Docker**：20.10+（如需部署真实长安链节点）

## 2. 环境变量配置

### 2.1 配置文件位置

后端环境变量配置位于：`backend/.env`

### 2.2 配置项说明

```env
# 运行模式：DEMO_MODE（演示模式，文件存储）或 CHAIN_MODE（真实链上）
RUN_MODE=DEMO_MODE

# 长安链 RPC 地址（CHAIN_MODE 时使用）
CHAIN_RPC_ADDR=http://localhost:12301

# 链 ID
CHAIN_ID=chain1

# 组织 ID
CHAIN_ORG_ID=TestCMorg1

# 合约名称
CHAIN_CONTRACT_NAME=did-contract
```

### 2.3 配置文件示例

参考文件：`backend/.env.example`

## 3. Node.js 环境安装

### 3.1 Windows 安装

1. 访问 https://nodejs.org/
2. 下载 LTS 版本安装包
3. 运行安装程序，按提示完成安装

### 3.2 验证安装

```bash
node --version
npm --version
```

## 4. Docker 环境安装（可选）

### 4.1 Windows 安装

```powershell
# 使用提供的脚本安装
cd Scripts
.\install-docker.ps1
```

或手动下载 Docker Desktop：https://www.docker.com/products/docker-desktop

## 5. 后端服务部署

### 5.1 安装依赖

```bash
cd backend
npm install
```

### 5.2 启动服务

```bash
# 开发模式
node app.js
```

### 5.3 服务验证

访问：http://localhost:3000/health

预期响应：
```json
{
  "status": "ok",
  "message": "DID System Service Running"
}
```

## 6. 长安链节点部署（CHAIN_MODE）

### 6.1 使用 Docker Compose 启动

```bash
cd Scripts
docker-compose -f docker-compose-chainmaker.yml -p blockchain up -d
```

### 6.2 验证链节点

容器状态检查：
```bash
docker ps
```

预期运行的容器：
- chainmaker-node1
- chainmaker-vm
- redis

## 7. 常见问题

### 7.1 端口被占用

**问题**：启动时提示 `Error: listen EADDRINUSE: address already in use :::3000`

**解决方案**：
- 修改 `backend/app.js` 中的端口号
- 或关闭占用 3000 端口的程序

### 7.2 数据丢失/重置

删除数据文件后重启服务：
```bash
# 删除 DEMO_MODE 数据
Remove-Item data\*.json
```

### 7.3 npm install 失败

尝试使用国内镜像：
```bash
npm config set registry https://registry.npmmirror.com
npm install
```
